What is imgcomp for?

The program "imgcomp" is for capturing long term motion triggered timelaps photographs using a Raspberry Pi and camera module. It is for the purpose of unattended monitoring and surveilance.

Unlike the "motion" package, it is not intended to work likc a CCTV camera. Instead, imgcomp is intended to fulfill the function of wildlife cameras - the sort that is battery powered and strapped to a tree for weeks at a time to monitor what sort of wildlife is present. I have used such cameras but found that the passive infrared motion sensing of these camers would often miss events, especially in winter. Sometimes they would even miss a tractor driving past the camera within five meters!

What imgcomp does not do

  • A raspberry Pi running imgcomp cannot be run off batteries (too much power draw)
  • Imgcomp does not capture video
  • Imgcomp has no user interface to speak of
  • Imgcomp does not work with USB web cameras
  • Imgcomp is not for people unfamiliar with using Linux from the command line.

    If you don't know how to edit configuration file, before asking me questions, get very familiar with linux. This might take you a few months. There is plenty about linux on the internet, please don't ask me for pointers.

    Compiling imgcomp

    It is assumed that you have rapsian already installed on your Raspberry Pi.

    I do not have a binary distribution for imgcomp. Before using imgcomp, you will have to compile it.

    Imgcomp uses libjpeg. Because I don't like external dependencies, I included a stripped down version of libjpeg in the distribution. You will have to compile this first:

    cd libjpeg
    make
    cd ..

    This will take a minute. Then go back into the main imgcomp directory and type:

    mkdir obj
    make

    This will compile imgcomp.

    Setting up imgcomp to run on the raspberry pi

    Enable camera module

    Run raspi-config to enable the camera module

    You also need to make sure that your camera module is installed and functioning. Take some pictures with raspistill to test. Type:

    raspistill -o foo.jpg"

    Then check that foo.jpg is an image of what the camera was pointed at.

    Make a ram disk

    Imgcomp uses raspistill to capture images to the file system. To avoid wearing out the flash card, it's best to capture to a ramdisk. Make a 20 megabyte ramdisk by adding the following line to the end of
    /etc/fstab
    :
    tmpfs /ramdisk tmpfs nodev,nosuid,size=20M 0 0

    Setting up Camera LED blinking

    imgcomp can optionally blink the camera module's LED (the one that comes on when the camera is active). If you wish the LED to blink when imgcomp detects motion, plese follow the following steps: Disable normal function of the camera LED, add the following line to the end of the file at
    /boot/config.txt
    disable_camera_led=1
    More on how to do this here

    imgcomp runs a separate program with superuser priviledges to blink the LED. This program is typically specified as "blinkc_camera_led" and is compiled along with imgcomp. You can test that this blinks the camera's LED by running blink_camera_led with root priviledges with this command (which will promot you for your password)

    sudo ./blink_camera_led

    blink_camera_led toggles the GPIO line that goes to the camera LED on the raspberry Pi model b+. If you are using a model A, you will have to modify the file blink_camera_led.c to specify the appropriate LED line, then recompile by typing "make".

    Once you have made sure that blink_camera_led blinks the LED, you will have to change the owner of the file to root, and set the "setuid bit". This will allow the program to run with administrator proviledges even when invoked by a regular user.

    sudo chown root blink_camera_led
    sudo chmod +s blink_camera_led

    After this, type "ls -l". The blink_camera_led file should now be highlighted because it runs with elevated priviledges.

    -rwsr-sr-x 1 root pi 6702 Jun 11 16:13 blink_camera_led

    Disable on screen logo and screen blanking

    These steps are not necessary, but I prefer to disable the raspberry pi logo at the top of the screen (more room for text) and disabling screen blanking: In the file /etc/kbd.config change:
    BLANK_TIME=30
    to
    BLANK_TIME=0
    Also change:
    POWERDOWN_TIME=30
    to
    POWERDOWN_TIME=0
    see here
